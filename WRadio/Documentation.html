Abstract


We live in the age of internet and internet radio is becoming more and more popular. Over the last few years, with the growth of technology and streaming media, internet radio has emerged as an excellent platform for providing wide range of choices to user from music, news,commentry to live talk shows. WRadio is a webcasting application that hosts internet radio stations from all around the world. The application pulls the data using an open source API provided by Dirble. It lets you select and stream from thousands of internet radio stations freely available over internet supported by dirble API.

Introduction


WRadio plays audio from internet radio streams provided by the API. For each station it displays the country from which that radio station is broadcasting along with the flag of the country. Also, when a radio station is selected for playing, the category of the music the station is playing is displayed. It also shows the bit-rate of audio the particular station is playing. If the station provides any description, the description will be displayed on the clicked radio stations.



Design of WRadio


WRadio has been designed using HTML5, CSS3, JavaScript and jQuery only. The data source is an API provided by internet radio hosting website called Dirble. Apart from the basic layout of each page, all the API data rendering on the webpage has been handled dynamically using jQuery and styled using CSS.

Dirble API:


WRadio makes an AJAX API call to dirble server using an API key. The API key provides access to station data according to categories/Genre. The data response received as a result of API call is in the format of JSON. jQUery code written in WRadio parses the JSON data received and process the relevant fields to be rendered on the webpage.

Country Data:


The API response contains the country data in the form of two character ISO code. jQuery code written in WRadio then, uses this country code and makes another AJAX call to a text file stored on the server containing delimited ISO country codes and country names along with the continent names. The country data pulled from this text file is then parsed and the respective country name is extracted to render on the web page.

Country Flags:


The station API response had low resolution country flags. The country flags on WRadio are pulled from another website GeoNames. The two character country code returned in API response is concatenated with GeoNames flags link to get the respective country flag dynamically for each station. jQuery concatenates the country code to the GeoName flag link to get the high resolution flags. The height and width of the flags has been handled using CSS.

Single Station Display:


After clicking play on a particular station from a list of stations, a <div> overlay is displayed for each radio station. The single station <div> has been designed completely using CSS and the appearance and closure have been handled using jQuery and CSS.



Features


  • Random Stations: Each time the home page of the application is loaded, a set of random stations is retreived through API call and displayed on the screen. jQuery code creates a random number within the range of station pages and retrieves a set of stations.
  • Genre List: User can click the Genre link from menu to get the list of Genre's available. The Genre list is generated dynamically through the API call. Genre list is populated using API call in order to get the up to date Genre list from Dirble.
  • Station details: A radio station when played, displays an image of the station if available, the website link to the broadcasting radio station, stream URL, bitrate of the audio being broadcasted, country from where broadcasting is taking place, HTML5 audio controls and the description of the station as to what kind of music the station plays If a particular station is down, an alert is displayed to notify the end user that the station is down.
  • LightBox effect: On selecting a particular radio station to be played, the selected station is displayed and the rest of the screen darkens. This effect has been implemented using jQuery and CSS overlay. On playing a station, the single station panel is generated dynamically and the hidden overlay is activated such that the radio station panel appears on top of the screen darkening the rest of the screen.
  • Search Bar: The search bar lets user search any string that user want. The user can enter country, station and cateogory name in the search bar. The search bar will send a request to Dirble API and the matching results will be rendered on the screen.
  • Pagination: The Genre page and Search page provides the user with the option of navigating the pages using pagination. The pagination has been implemented in jQuery. It provides the user ease of navigating through different pages. The API returns maximum of 30 stations per page hence in successive pages the number of stations might vary.


Future Scope


  • Feature to display and play the music stations by Countries and Continents. By extending the API call and manipulating the data received, it is possible to display the stations by country and continents.


  • History of the music or programs broadcasted by the selected station.
